""" Custom datatset loader
    based on https://github.com/utkuozbulak/pytorch-custom-dataset-examples
"""
import pandas as pd
import imageio

class SimpleDataset():
    def __init__(self, data_path, csv_name, transform = None ):
        """
        Args:
            data_path (string): path to the folder where images and csv files are located
            csv_name (string): name of the csv lablel file
            transform: pytorch transforms for transforms and tensor conversion
        """
        # Set path
        self.data_path = data_path
        # Read the csv file
        self.data_info = pd.read_csv(data_path + csv_name, header=None)
        # First column contains the image paths
        self.image_arr = np.asarray(self.data_info.iloc[:, 0])
        # Second column is the labels
        self.label_arr = np.asarray(self.data_info.iloc[:, 1])
        # Calculate len
        self.data_len = len(self.data_info.index)
        
    def __getitem__(self, index):
        # Get image name from the pandas df
        single_image_name = self.image_arr[index]
        # Open image
        img_as_img = imageio.imread(self.data_path + single_image_name)
        
        # Get label(class) of the image based on the cropped pandas column
        single_image_label = self.label_arr[index]
        
        return (img_as_img, single_image_label)

    def __len__(self):
        return self.data_len

KNN Observations¶

The K Nearest Neighbor algorithm is a supervised learning algorithm useful in both classifier and regression type of problems. It is expected that as K increases, the predictions become more stable. However, 'stable' does not indicate accuracy as evident from the exercise. The highest scores were obtained for K=1. The KNN algorithm becomes significantly slower as the number of training and testing entries increases.

Another interesting observation is the split between training time and testing time. In general, we would like any classifier to take more time to train and less time to test so that in the real world, we could deploy a pre-trained model which makes quick predictions. KNN, however, takes less time to train and significanty more time to make predictions. This is because in the training step, KNN simply memorizes all the data available. The actual computaions take place in the testing stage.

Fewer number of training (25000) and testing (1500) samples had to be used to bring down the runtime to within 10 minutes. This could affect the performance of the classifier because the subset of training and testing examples chosen is not guaranteed to be representative of the actual set i.e frequency of some characters may be more than others, causing the weights to be unbalanced.

MLP Observations¶

The multilayer perceptron classifier is a classifier model that optimizes a log-loss function using stochastic gradient decent. It trains iteratively since at each time step the partial derivatives of the loss function with respect to the model parameters are computed to update the parameters. A few important parameters to note are alpha - L2 penalty - which acts as a regularization term and the initial learning rate which affects convergence.

In this exercise, the classifier performance score on the MNIST dataset was found to be highly sensitive to the initial learning rate. Several values of initial learning rate were tried and the best results were observed for values of the order of 0.0002. Different values of max_iterations (upto 500) were also tried but convergence could not be achieved. The max_iterations were set to 25 since the log-loss function had reduced to acceptable limits withing 25 iterations.

Hidden layers used were of size 100. The MLP classifier performed poorly on the small dataset compared to all other classifiers.

SVM Observations¶

Support vector machines create hyperplanes in high-dimentional feature space for classification. SVMs are generally faster because it uses kernel functions but it is at the cost of generalization error which increases with dimentionality. The effectiveness of SVM depends on the selection of kernel, the kernel's parameters, and soft margin parameter C.

The parameters provided in the lab tutorial for linear classifier and cubic polynomial performed reasonably well on both the MNIST dataset and our own test dataset. For the radial basis function, the parameters given in the lab tutorial performed extremely poorly. The radial basis function classifier also took considerably more time to run than the other two for the same training and testing set sizes.

The training and testing set sizes were further reduced to bring down the runtime of radial basis function classifier to within 10 minutes. After trying out several differnt combinations of parameters, it was obsereved that gamma=0.01 improved the performance of the classifier on MNIST from 0.112 to 0.821 and that on the small dataset from 0.0833 to 0.4167.
